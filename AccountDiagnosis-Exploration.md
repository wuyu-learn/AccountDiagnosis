## 4. 两种数据取值方式（dataMode）

### Inline（直出/内联）

OpenClaw 直接输出完整 `data`，前端**直接渲染、不二次取数**。

```json
{
  "card_id": "account_total_asset_card",
  "dataMode": "inline",
  "data": { "total_asset": 1432420.58, "today_profit": 3520.12, ... }
}
```

**适用**：数据量小、无复杂交互、无长时序走势的汇总/列表卡。  
**例子**：ACC-01、ACC-02、ACC-04、ACC-06~ACC-10、ACC-18。

### Deferred（延迟/异步）

OpenClaw 仅输出 `dataMode: 'deferred'` + `data.request`（标题 + 参数），前端再调专用 API 取数后渲染；取数失败则**整卡隐藏**。

```json
{
  "card_id": "fund_holding_detail_card",
  "dataMode": "deferred",
  "data": {
    "request": { "account_type": "fund", "card_id": "acc03", "fund_code": "000001" }
  }
}
```

前端再调用：

```http
POST /api/account/card/acc03
# body 透传 data.request
```

**适用**：数据量大、多 tab/维度切换、长时序走势图、需穿透计算的卡。  
**例子**：ACC-03、ACC-05、ACC-11~ACC-17。

### 设计意图推测

| 维度 | Inline | Deferred |
|------|--------|----------|
| 响应速度 | 快，一次返回 | 慢一点，需二次请求 |
| 大模型成本 | 数据直接塞进回答，占用 token | 只输出参数，省 token |
| 计算复杂度 | 只能承载简单计算 | 复杂聚合、走势图交给后端 |
| 失败处理 | 数据已存在，一般能渲染 | 失败则整卡隐藏 |
| 前后端职责 | 后端/编排层准备好一切 | 后端提供专用接口，前端按需拉取 |

> 核心原则：**每张卡的 dataMode 是固定的，不因"被怎么打开"而改变。** 入口（直接问 / 点击下钻）只决定何时触发，不改取数方式。

## 5. 下钻关系网

```text
ACC-01 账户总资产 ＝ 顶层入口
├── 货币基金格 → ACC-07 货币账户详情 → ACC-08 货币持有详情（终点）
├── 基金格 → ACC-02 基金账户详情 → ACC-03 基金持有详情
├── 投顾格 → ACC-04 投顾账户详情 → ACC-05 投顾组合详情 → ACC-06 投顾调仓记录（终点）
├── 组合格 → （815 置灰 / 即将上线）
└── 专户格 → ACC-09 专户账户详情 → ACC-10 专户持有详情（终点）

ACC-11~17（持仓明细 / 资产结构 / 收益批）为独立入口，非 ACC-01 下钻。
```

## 7. 关键交互规则与红线

- **跳过规则**：图表/明细卡数据不足时**整卡跳过不渲染**；非图表卡字段缺失显示 `—`（字段降级，卡片仍渲染）。
- **诊断不开方**：账户诊断卡只分析、不给调仓/买卖建议；调整类问题通过 `guidance` 引导至策略配置 Agent。
- **估值四态**：`estimated`（盘中预估）、`updating`（更新中）、`confirmed`（盘后确认）、`hidden`（仅展示净值日期）。
- **收益口径一致**：同一卡内头部与明细的收益栏措辞/时间点必须一致。
- **持有 vs 累计**：持有收益 = 当前持仓相对成本；累计收益 = 建仓以来全口径（含已落袋）。
- **专户不穿透**：专户底层不可展示，`can_lookthrough` 强制 `false`。
- **货基恒正**：货币基金收益不为负，不染绿。
- **投顾内容只读**：组合报告、调仓说明、主理人观点均为原文摘录，平台不二次加工、不含交易入口。
- **去除同期基民对比**：收益总览卡不允许"战胜 X% 基民/同类排名"。
- **安抚话术属实才说**：亏损态只基于真实数据安慰，长期为负不硬安慰。
- **ACC-24 综合诊断卡已废弃**：编号暂不复用；"我账户怎么样"由多卡摘要模式承接，不生成健康分或综合评分。

## 8. 主要数据源 / API

| 数据源 | 用途 |
|--------|------|
| `get_account_asset_summary` | 总资产、账户类型资产、资产来源 |
| `get_asset_structure` | 资产结构五维度、持仓明细结构 |
| `get_holdings` | 持仓列表（基金/投顾/专户/货币 tab） |
| `get_holding_detail` | 单只基金/货币/专户持有详情 |
| `get_return_performance` | 收益总览、收益日历、收益构成、走势图 |
| `get_return_attribution` | 收益归因（按产品/资产类别/行业） |
| `get_advisor_rebalance` | 投顾组合调仓记录 |
| `get_watchlist` | 自选基金列表 |
| `get_fund_valuation` | 基金估值/净值/涨跌 |
| `get_related_sector` | 关联板块涨跌 |

**deferred 统一接口**：

```http
POST /api/account/card/{cardId}
```

例如 `/api/account/card/acc03`，入参透传 `data.request`，出参与 inline 卡同构。失败处理：HTTP 非 2xx 或超时 → 该卡整卡隐藏。

## 9. 特殊状态与边界情况

| 类别 | 关键规则 |
|------|----------|
| 估值状态 | 盘中预估标"预估"；盘后确认标"已更新 · 净值 06-12"；QDII 标"净值 06-11"等披露日。 |
| 数据不可用 | 主数字显示 `——`，不显 `0`，提供刷新重试；账户类卡片仍渲染。 |
| 空持仓 | 基金/投顾等账户无持仓时，正面陈述"暂无在持"，不渲染空表。 |
| 货基/QDII 特殊 | 货基周末/节假日有收益则照常红/绿，仅真正无收益为灰；QDII 不按境内盘中预估口径，用昨日结算价。 |
| 连涨连跌徽标 | 基于**确认净值**累计连续同向天数，≥2 才显示；盘中预估不改变计数。 |
| 对账闭环 | 各来源收益加总 = 区间总收益；组合市值合计 = 投顾账户市值；日/月/年累计与收益总览同源。 |
| 归因方向不可反 | `contribution_direction` 取 MCP 原值，正红负绿，前端不可反向或自算。 |
| TWR vs MWR 口径 | 走势图与业绩比较基准对比用份额归一（TWR）；区间收益数字/体验差解释用资金加权（MWR）。 |
| 无自选 | 自选估值卡空态引导添加自选，不展示五角星等冗余开关。 |
