# ACC-11 · holding_list_card 持仓明细卡

持仓明细卡用于回答用户"我都买了哪些 / 每只怎么样 / 看持仓"，展示账户下各产品的持仓情况，包括每只产品的市值、收益、占比及净值/估值信息，并支持按产品视角与按基金穿透视角切换查看。

卡片以账户为统计范围，汇总货币、基金、投顾、组合、专户等产品的持仓数据。按产品视角下各产品类型展示深度不同：基金最完整、投顾可穿透、组合可穿透、专户仅汇总、货币最精简。按基金穿透视角将货币、基金、投顾组合按基金维度打平合并，同基金跨来源合并展示。

数据源：`get_holdings` + `get_asset_structure`

卡片 `dataMode`：`deferred`

---

## 1. 数据相关

### 1.1 数据统计范围

持仓明细卡统计用户全账户的持仓数据，覆盖：

- 货币
- 基金
- 投顾
- 组合
- 专户

- 按产品视角：以产品为最小展示单位，不同类型产品展示字段不同。
- 按基金穿透视角：将货币、基金、投顾组合中的底层基金按基金维度打平合并，同基金跨来源合并；专户不纳入穿透。
- 金额单位统一为人民币元。

### 1.2 持仓总市值

- 持仓总市值 = 各产品（或穿透后各基金）持仓市值之和。
- 按产品视角下，总市值按产品类型分层加总。
- 按基金穿透视角下，总市值为穿透合并后的基金市值合计，与按产品视角下的基金+货币+投顾部分总市值一致。

### 1.3 当日预估收益

- 当日预估收益统计账户在当前收益日产生的收益。
- 当日预估收益支持真实收益与预估收益并存：
  - 今日收益 = 今日已更新收益 + 今日预估收益
  - 今日已更新收益：已经取得真实收益数据的产品收益合计。
  - 今日预估收益：尚未取得真实收益、当前使用预估值的产品收益合计。
- 同一个产品的真实收益和预估收益不得重复计算。
- 产品存在真实收益时，优先使用真实收益。
- 产品不存在真实收益时，使用预估收益。
- 产品既没有真实收益也没有预估收益时，不参与当日预估收益计算。
- 当日预估收益率 = 当日预估收益 / 上一收益日账户总资产。

### 1.4 数据状态

当日预估收益数据分为以下状态：

| 状态 | 枚举值 | 判断口径 |
|------|--------|----------|
| 全部预估 | estimated | 参与统计的产品全部使用预估收益 |
| 更新中 | mixed | 真实收益与预估收益并存 |
| 全部更新 | real | 参与统计的产品全部使用真实收益 |
| 暂无数据 | unavailable | 当前没有可用的真实收益或预估收益 |

- 全部预估：今日已更新收益 = 0，今日预估收益存在。
- 更新中：今日已更新收益存在，今日预估收益存在。
- 全部更新：今日预估收益 = 0，今日已更新收益存在。
- 暂无数据：今日已更新收益不存在，今日预估收益不存在；金额为 0 不等于暂无数据。

### 1.5 产品类型展示规则

#### 基金

- 展示最全字段。
- 冻结列：产品简称 + 产品代码。
- 横滑列：最新价/估值、最新净值、持有收益、持仓/占比等。
- 列头可排序，当前排序列高亮。
- 支持左右滑动查看更多列、上下滑动查看全部、点击单只看详情。

#### 投顾

- 组合层列示，不展示份额（以管理人披露为准）。
- 单个投顾组合可展开查看底层基金持仓。
- 底层基金以缩进 + 浅底区分层级。
- 组合市值合计与投顾账户卡一致。

#### 组合

- 组合层列示，可展开底层基金持仓。
- 展示逻辑参考投顾 Tab。

#### 专户

- 按监管要求仅展示汇总信息，不穿透展示底层持仓。
- 以纵向汇总卡形式展示，非横滑表。
- 专户为预估市值，以管理人披露为准。

#### 货币

- 最简展示。
- 展示核心字段：产品名称、代码、市值、当日收益等。

### 1.6 按基金穿透规则

- 将货币、基金及投顾组合按基金维度打平合并。
- 同一只基金跨来源（直接持有 / 投顾组合内）已合并。
- 合并后展示"来源"列，标识该基金来自"汇添富持仓"或"外部导入"等。
- 专户不参与穿透。
- 穿透视角下不再区分产品类型 Tab，统一以基金列表展示。

## 2. 交互逻辑与展示

### 2.1 视角切换

- 卡片顶部提供视角切换器：
  - 按产品
  - 按基金穿透
- 默认选中"按产品"。
- 切换视角后，下方内容区域刷新，Tab 栏与列表形态相应变化。

### 2.2 产品类型 Tab

- 按产品视角下，展示五类 Tab：
  - 基金
  - 投顾
  - 组合
  - 专户
  - 货币
- 每个 Tab 显示该类型产品数量。
- Tab 高亮当前选中类型。

### 2.3 列表交互

- 基金 Tab 支持：
  - 冻结首列（产品简称 + 代码），其余列可横向滑动。
  - 列头点击排序，当前排序列高亮。
  - 点击单行进入产品详情。
- 投顾 / 组合 Tab 支持：
  - 点击组合卡片展开 / 收起底层基金。
  - 展开后显示"展开其余 N 只"入口（当基金数量较多时）。
- 专户 Tab 不支持穿透，仅展示汇总卡片。
- 货币 Tab 保持极简列表。

### 2.4 头部信息

- 卡片头部展示：
  - 标题：持仓明细
  - 数据状态徽标：如"预估 14:25"
  - 持仓总市值（元）
  - 当日预估收益及收益率
  - 说明文案：本页涨幅、收益、市值、占比均为盘中预估，以披露净值为准
- 按基金穿透视角下头部展示穿透后的总市值与当日预估收益。

## 3. 模块字段

### 3.1 入参 Request

数据源：`get_holdings` + `get_asset_structure`

| 卡片元素 | 字段 | 类型 | 必填 | 说明 |
|----------|------|------|------|------|
| 统计范围 | scope | string | 是 | 固定传 `全账户` |
| 视角类型 | view_type | string | 是 | `by_product`（按产品）或 `by_fund`（按基金穿透） |
| 产品类型 | product_type | string | 按产品视角必填 | `fund` / `advisor` / `combo` / `separate` / `money` |
| 排序字段 | sort_field | string | 否 | 当前排序列字段名 |
| 排序方向 | sort_order | string | 否 | `asc` 或 `desc` |
| 是否穿透 | include_underlying | boolean | 否 | 按产品视角下是否返回投顾/组合底层基金，默认 `true` |

### 3.2 出参 Response

Response 分为：

- 基础信息
- 持仓汇总
- 产品持仓列表
- 按基金穿透列表
- 投顾/组合底层基金（展开时）

#### 基础信息

| 卡片元素 | 字段 | 类型 | 说明 |
|----------|------|------|------|
| 当前视角 | view_type | string | `by_product` 或 `by_fund` |
| 当前产品类型 | current_product_type | string | 按产品视角下当前选中的 Tab 类型 |
| 数据日期 | data_date | string | 当前持仓数据对应日期，格式：YYYY-MM-DD |
| 数据更新时间 | updated_at | string | 数据最近更新时间 |
| 总市值 | total_market_value | number | 当前视角下的持仓总市值，单位：元 |
| 当日预估收益 | daily_estimated_profit | number | 当前视角下的当日预估收益，单位：元 |
| 当日预估收益率 | daily_estimated_return_rate | number/null | 当日预估收益 / 上一收益日总资产 |
| 今日数据状态 | today_data_status | string | `estimated` / `mixed` / `real` / `unavailable` |
| 上一收益日总资产 | previous_total_asset | number/null | 当日预估收益率的计算分母 |

#### 持仓汇总

对象字段：`holding_summary`

| 卡片元素 | 字段 | 类型 | 说明 |
|----------|------|------|------|
| 产品总数 | total_count | number | 当前视角下产品/基金总数 |
| 货币数量 | money_count | number | 货币基金数量 |
| 基金数量 | fund_count | number | 基金数量 |
| 投顾数量 | advisor_count | number | 投顾组合数量 |
| 组合数量 | combo_count | number | 组合数量 |
| 专户数量 | separate_count | number | 专户数量 |

#### 产品持仓列表

数组字段：`holdings`

| 卡片元素 | 字段 | 类型 | 说明 |
|----------|------|------|------|
| 产品ID | product_id | string | 产品唯一标识 |
| 产品名称 | product_name | string | 产品简称 |
| 产品代码 | product_code | string | 产品代码 |
| 产品类型 | product_type | string | `fund` / `advisor` / `combo` / `separate` / `money` |
| 最新价/估值 | latest_price | number/null | 基金最新价或估值 |
| 最新价日期 | latest_price_date | string/null | 最新价对应日期 |
| 最新净值 | latest_nav | number/null | 基金最新净值 |
| 最新净值日期 | latest_nav_date | string/null | 最新净值对应日期 |
| 持有收益 | holding_profit | number | 该产品持仓收益，单位：元 |
| 持有份额 | holding_quantity | number/null | 持有份额；投顾/组合层不展示 |
| 持仓市值 | holding_market_value | number | 该产品持仓市值，单位：元 |
| 持仓占比 | holding_ratio | number | 占当前视角总市值比例 |
| 当日收益 | daily_profit | number | 当日收益，单位：元 |
| 当日收益率 | daily_return_rate | number | 当日收益率 |
| 是否可展开 | is_expandable | boolean | 投顾/组合是否可展开底层基金 |
| 底层基金数 | underlying_count | number | 投顾/组合包含的底层基金数量 |
| 底层基金 | underlying_funds | array | 展开时返回的底层基金列表 |

#### 按基金穿透列表

数组字段：`penetrated_holdings`

| 卡片元素 | 字段 | 类型 | 说明 |
|----------|------|------|------|
| 基金ID | fund_id | string | 基金唯一标识 |
| 基金名称 | fund_name | string | 基金简称 |
| 基金代码 | fund_code | string | 基金代码 |
| 当日收益 | daily_profit | number | 合并后当日收益，单位：元 |
| 持仓市值 | holding_market_value | number | 合并后持仓市值，单位：元 |
| 来源 | source | string | 如 `汇添富持仓`、`外部导入` |
| 来源明细 | source_details | array | 该基金各来源明细 |

#### 投顾/组合底层基金

数组字段：`underlying_funds`

| 卡片元素 | 字段 | 类型 | 说明 |
|----------|------|------|------|
| 基金ID | fund_id | string | 底层基金唯一标识 |
| 基金名称 | fund_name | string | 底层基金简称 |
| 基金代码 | fund_code | string | 底层基金代码 |
| 持仓市值 | holding_market_value | number | 在该组合中的持仓市值，单位：元 |
| 持仓占比 | holding_ratio | number | 占该组合市值比例 |

### 3.3 数据说明

- 收益金额带正负号。
- 收益率使用小数返回，例如 `0.248` 表示 24.80%。
- 当 `previous_total_asset` 为 0 或缺失时，`daily_estimated_return_rate` 返回 `null`。
- 金额等于 0 不代表没有数据；接口需要区分"已计算且结果为 0"与"尚未产生数据"。
- 同一个产品不得同时计入当日已更新收益和当日预估收益。
- 产品已有真实收益时，真实收益覆盖该产品的预估收益。

---

## 4. 边界与异常

- 数据不可用态：主数字显"——"不显 0/空白，给刷新入口；卡片仍渲染（账户类必出）。
- 按基金穿透时，若某基金仅来源于外部导入，来源列展示"外部导入"。
- 投顾/组合底层基金按管理人披露为准，组合层不展示份额。
- 专户产品按监管要求仅展示汇总信息，不穿透展示底层持仓。
