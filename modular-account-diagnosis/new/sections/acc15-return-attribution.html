字段与交互逻辑
  • 方向不可反（最关键红线）：贡献方向（正/负）与数值取 MCP contribution_direction 原值，前端不可反向、不自算；盈利红 #F4333C/拖累绿 #22A875——搞反等于把赚的说成亏的
  • 周期下拉
    今日/昨日/本月/本年/历史以来；默认本年，切换后重算总收益与贡献列表
  • 单向分组条：盈利组/拖累组分区、组内归一占满宽度（窄屏友好）；量大各取Top10
  • 总账：各项贡献加总≈总收益（措辞"约等于"）
  • 只陈述不建议：不给调仓建议，调整引导策略配置；归因的正负即盈亏，用涨跌红绿恰当
正确性红线：contribution_direction 取 MCP 原值，正红负绿不可反、不自算。这是收益归因最关键的正确性校验。
